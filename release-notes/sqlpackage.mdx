SqlPackage is a cross-platform command-line utility that automates database development tasks such as publish, extract, export, import, and deploy report. You can install it as a .NET tool from [NuGet (Microsoft.SqlPackage)](https://www.nuget.org/packages/Microsoft.SqlPackage). SqlPackage version numbers align with the underlying DacFx library version numbers.

<Note>
SqlPackage version numbers match the corresponding Microsoft.SqlServer.DacFx NuGet package version. For example, SqlPackage `170.4.43-preview` uses DacFx `170.4.43-preview`.
</Note>

### Added

- Utilize bcp files to export data that includes unsupported PolyBase types such as `sql_variant`, UDT, CLR UDT, and tables with columns containing LOB data larger than 1 MB. This enables a Parquet and BCP hybrid approach to manage unsupported PolyBase types and limitations.

</Accordion>

<Accordion title="170.0.57-preview">

### Added

- Adds support for extracting table data with Parquet format for tables with column types `timestamps`, `rowversions`, `ntext`, `text`, `image`, `json`, `xml`, and `vector`.

- Fixes [#453](https://github.com/microsoft/DacFx/issues/453).
