DacFx and SqlPackage are Microsoft's suite of tools for SQL database development and deployment. Store your database schema as code in SQL projects, compile it to a portable `.dacpac` package, and deploy it consistently across SQL Server, Azure SQL, Azure Synapse, and Microsoft Fabric environments.

## Get started

<Steps>
  <Step title="Install SqlPackage and the SDK templates">
    Install the SqlPackage CLI tool and the SQL project templates via the .NET CLI.

    ```bash
    dotnet tool install -g microsoft.sqlpackage
    dotnet new install Microsoft.Build.Sql.Templates
    ```
  </Step>
  <Step title="Create a SQL project">
    Scaffold a new SQL project using the `sqlproj` template.

    ```bash
    dotnet new sqlproj -n MyDatabase
    ```
  </Step>
  <Step title="Add your database objects">
    Create `.sql` files with your T-SQL table, view, and stored procedure definitions. The SDK automatically includes all `*.sql` files during build.
  </Step>
  <Step title="Build and publish">
    Compile to a `.dacpac` and deploy to your target database.

    ```bash
    dotnet build
    sqlpackage /Action:Publish /SourceFile:"bin/Debug/MyDatabase.dacpac" \
      /TargetConnectionString:"<your-connection-string>"
    ```
  </Step>
</Steps>

## Key concepts

<AccordionGroup>
  <Accordion title="What is a dacpac?">
    A `.dacpac` (Data-tier Application Package) is a ZIP-format file that contains a database schema model, pre/post-deployment scripts, and metadata. It is the compiled output of a SQL project and the input for SqlPackage deployment operations.
  </Accordion>
  <Accordion title="What is a bacpac?">
    A `.bacpac` file captures both the schema and data of a database, suitable for portability and archiving. SqlPackage's Export and Import actions work with `.bacpac` files.
  </Accordion>
  <Accordion title="Which SQL platforms are supported?">
    SQL projects and SqlPackage support SQL Server 2012 through 2025, Azure SQL Database, Azure SQL Managed Instance, Azure Synapse Analytics, Azure Synapse Serverless, and SQL database in Microsoft Fabric.
  </Accordion>
  <Accordion title="How does schema drift work?">
    SqlPackage compares the schema in your `.dacpac` against the live database and generates an incremental deployment script — only altering objects that differ. You never write migration scripts by hand.
  </Accordion>
</AccordionGroup>
